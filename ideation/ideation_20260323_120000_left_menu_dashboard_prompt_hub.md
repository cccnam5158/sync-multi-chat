# Ideation: 왼쪽 메뉴 재구성 · Dashboard · Prompt Hub · Slash 개선

> 원안: `feat_left_menu_renewal_and_prompt_hub/draft_ideation_260323.md`  
> 작성일: 2026-03-23

---

## 0. 목적과 범위

Sync Multi Chat에서 **프롬프트를 카테고리·태그로 체계적으로 관리**하고, **자주 쓰는 프롬프트를 빠르게 찾으며**, **이용 현황·대화 이력 탐색**을 쉽게 하는 것이 목표다.

- 범위: 왼쪽 사이드바 내비게이션, Dashboard 화면, Prompt Hub 화면, Custom Prompt Builder 확장, Chat History 보강, 메인 입력 `/` Slash Command UX 개선.
- 비범위: AI 서비스별 채팅 URL/로그인 로직 변경, 백엔드 서버 연동(로컬 저장 유지).

---

## 1. 문제 정의 (Pain Points)

1. **프롬프트 묶음 관리 부재**: 연관 프롬프트를 카테고리로 묶어 관리할 수 없다.
2. **Slash 탐색 비효율**: `/` 후 전체 목록이 길고, 입력이 컨텍스트 메뉴와 겹쳐 가독성이 떨어진다.
3. **이용·이력 파악 어려움**: 대화 세션 수·시간대별 패턴을 보기 어렵고, 특정 제목으로 빠르게 찾기 어렵다.

---

## 2. 가치 가설

- **2-depth 카테고리 + 태그 + 즐겨찾기**로 프롬프트 라이브러리를 정리하면 검색·재사용 시간이 줄어든다.
- Slash 메뉴 **전용 검색 필드 + 제목 하이라이트 + 즐겨찾기 상단**이면 자주 쓰는 템플릿 선택이 빨라진다.
- **Dashboard**로 모드·프롬프트·히스토리 요약을 한 화면에서 보면 앱 사용 맥락이 명확해진다.

---

## 3. 왼쪽 메뉴 정보 구조 (IA)

### 3.1 내비게이션 항목 (순서 고정)

| 항목 | 설명 |
|------|------|
| **+ New Chat** | 메인 영역 `New Chat`과 동일: 새 세션 생성, 웹뷰 기본 URL로 리셋. |
| **Dashboard** | 이용 현황 요약(아래 4절). |
| **Prompt Hub** | 프롬프트 허브(5절). |
| **Chat History** | 기존 히스토리 기능 유지 + 제목 검색·정렬·날짜 구간 그룹(6절). |

- 햄버거로 사이드바 접기/펼치기 유지. **Prompt Hub** 선택 시 작업 폭을 넓히기 위해 사이드바 **가로 확장** 허용.
- 접힘 상태에서는 핵심 아이콘만 표시해 동일 항목으로 이동 가능하게 한다.

### 3.2 Chat History 유지 조건

- **복수 선택 모드**, 하단 **All / Clear / Delete**, **Rename / Delete** 컨텍스트 메뉴, 무한 스크롤 로딩은 기존과 동등하게 동작해야 한다.
- 추가로 **제목 검색**, **정렬(최근 수정·생성·제목)**, **날짜 그룹(오늘 / 어제 / 지난 주 / 지난 달 / 그 이전)** 표시.

---

## 4. Dashboard 요구사항

- **현재 모드**: Single / Multi 및 선택된 AI 서비스(또는 Single 시 서비스·인스턴스 요약) 표시.
- **프롬프트 통계**: 사용자 정의 카테고리 개수(시스템 가상 카테고리 제외 규칙은 구현 스펙 참고), 저장된 커스텀 프롬프트 총 개수.
- **대화 이력**:
  - IndexedDB에 저장된 **총 세션 수**.
  - **날짜별 세션 수** 차트: 기간 단위 선택 **일 / 주 / 월 / 연**에 따라 버킷 집계(세션의 `updatedAt` 또는 `createdAt` 기준, 구현 일관성 유지).

---

## 5. Prompt Hub 요구사항

### 5.1 레이아웃

- **상단 바**: 좌측 `Prompt Hub (총 N개)`, 우측 `+ New Prompt` → Custom Prompt Builder(기존 CPB 모달)를 새 프롬프트 편집 상태로 연다.
- **본문**: 좌 **필터 영역**, 우 **콘텐츠 영역**(필요 시 상단 필터 / 하단 목록 2단 구성). 데스크톱 기준 1×2 그리드에 가깝게 배치.

### 5.2 필터

- **제목 검색**: 입력 시 목록 실시간 필터; 매칭되는 프롬프트가 속한 카테고리만 남기도록 카테고리 트리도 좁힐 수 있음(구현: 검색어가 있을 때 매칭 프롬프트의 카테고리 경로만 표시).
- **카테고리**: 최대 **2-depth** 트리에서 선택. 시스템 **미분류** 버킷 포함.
- **태그**: 프롬프트에 부여된 태그로 필터(다중 태그 AND 또는 OR는 구현에서 OR로 단순화 가능).

### 5.3 목록 표시

- **보기 모드**: **테이블** / **미리보기(카드)** 전환.
- 각 항목: **제목**, **간략 설명**, **본문 일부 미리보기**, **수정**(CPB), **복사**(클립보드), **복제**, **삭제**, **실행**(메인 입력에 넣고 전송 흐름은 CPB “Copy to Input”과 유사하거나 즉시 Send — 구현: 메인 입력에 삽입 후 사용자가 Send).
- 제목 또는 수정 클릭 시 해당 프롬프트로 **CPB 편집** 연다.

### 5.4 카테고리·데이터 규칙

- 카테고리 **최대 2-depth**; 별도 **카테고리 관리 모달**에서 추가/삭제(이름 변경은 동일 모달에서 처리).
- 시스템 **즐겨찾기** 가상 그룹: 구조는 사용자가 편집하지 않으며, 프롬프트 단위 **즐겨찾기** 및 “카테고리 전체를 즐겨찾기에 포함” 시 해당 카테고리 ID를 **즐겨찾기 카테고리 목록**에 넣는 방식으로 처리(구현 스키마 참고).
- 카테고리 미지정 프롬프트는 모두 **미분류**로 표시; **기존 마이그레이션 시** categoryId 없음 → 미분류.
- **다중 선택** 후 **한 번에 카테고리 지정** 가능.

### 5.5 Custom Prompt Builder 연동

- 편집 화면에서 **카테고리 선택**, **태그**, **간략 설명(summary)** 입력.
- Import/Export 시 카테고리 정의 및 확장 필드 포함 여부는 구현 단계에서 JSON 스키마 버전으로 관리.

---

## 6. Slash Command (`/`) 개선

1. 드롭다운 **상단에 검색 전용 입력** 추가. `/` 입력 직후 **포커스를 해당 입력으로 이동**해, 메인 텍스트와 겹치지 않게 필터링한다.
2. 검색어에 맞게 프롬프트 제목을 실시간 필터하고, 제목 텍스트에서 **매칭 구간 하이라이트**(라이브).
3. **즐겨찾기** 프롬프트(및 즐겨찾기에 포함된 카테고리 소속 프롬프트)를 목록 **최상단**에 둔다.

---

## 7. 수용 기준 (요약)

- 네 가지 내비 항목이 모두 동작하고, New Chat이 메인 버튼과 동일하게 새 세션·URL 리셋을 수행한다.
- Dashboard에 모드·프롬프트 수·히스토리 수·차트가 표시된다.
- Prompt Hub에서 필터·보기 전환·CRUD·벌크 카테고리 지정이 가능하다.
- CPB에서 카테고리·태그·요약·즐겨찾기를 저장할 수 있다.
- Chat History에 제목 검색·정렬·날짜 그룹이 보이고 기존 벌크 삭제 등이 깨지지 않는다.
- Slash 메뉴에 상단 검색·하이라이트·즐겨찾기 우선 정렬이 적용된다.
