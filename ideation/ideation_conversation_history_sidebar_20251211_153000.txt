새로운 기능을 개발하려고 해
아래 내용을 기반으로 @ideation 디렉토리 밑에 기존 파일의 네이밍과 컨텐츠 내용을 참고하여,
ideation을 구체화 (구현 세부 방안, 제약 사항, 극복 방안 검토 필수) 해서,
markdown 형식으로 @ideation 디렉토리 하위에 저장한 뒤, @Multi-AI-Chat-SRS-Unified.md 을 업데이트 한 다음 구현 진행해줘

이하는 새로운 기능 설명이야

일반적인 (예를 들면, ChatGPT, Claude 등) AI 서비스의 경우, 왼쪽 펼침과 접힘이 가능한 대화 히스토리를 보여주는 기능이 구현되어 있어
새로운 대화가 시작되면 왼쪽의 메뉴에 새로운 대화가 타이틀 요약과 함께 추가되는 구조이고, 
나중에 로그인했을 때, 최근 대화 이력을 선택해서 해당 대화부터 시작하는 기능이 있어

내가 만든 앱은 여러 AI 서비스에 동시에 프롬프트 및 파일을 자동으로 전송해서 대화하는 기능을 구현하고 있어
내가 앱을 기동한 후, 챗 프롬프트 입력란에 프롬프트를 입력하거나 파일을 첨부한 뒤,Ctrl+Enter를 누르거나, Send 버튼을 누르면 대화가 시작돼
그리고 "New Chat" 버튼을 클릭하면 새로운 대화가 시작되도록 각 웹 뷰를 새로운 대화로 시작하도록 여는 기능이 있어
그리고 챗 프롬프트의 설정 상태 (예를 들어, Anonymous 버튼, Scroll Sync, 레이아웃, AI 서비스 등)와 웹 뷰의 세션 URL주소와 현재 웹 뷰에 열려있는 모든 페이지의 주소를 기록해서
앱이 재기동되었을 때 해당 설정 정보를 기반으로 마지막 대화를 이어가도록 구현되어 있어 

기 구현된 기능을 최대한 활용해서 새로운 기능을 구현해줘

1. 앱의 왼쪽에 펼치고 접힘이 가능한 (펼침/접힘 버튼은 햄버거 아이콘) 대화 히스토리 조회 기능
2. 왼쪽의 햄버거 메뉴는 초기에는 접혀있는 상태이다
3. 햄버거 아이콘을 선택하면 대화 히스토리 목록이 펼쳐지고, 접혀진다.
4. 대화 히스토리 조회 화면은 최근 항목 순으로 정렬되어 표시된다. 
5. 처음 앱 기동시에는 접혀있는 상태이며, 마지막 최근 대화가 선택되고 화면 설정 상태가 복원된 상태로 앱이 기동된다 (현재 구현상태 유지)
   - 기능을 초기 구현 후, 앱 기동 시 DB를 생성하여야 하며, 현재 기능 구현된 웹 뷰 목록 및 챗 프롬프트 버튼 상태 정보를 최근 첫번째 대화로 저장하여 구동한다.
6. 챗 프롬프트의 "New Chat" 버튼이 클릭되면 대화 히스토리에 새로운 대화가 저장된다. 이 때 현재 웹 뷰 오픈 및 챗 프롬프트 상태 정보를 직전 대화 ( 즉, 현재 대화 목록에 반영한다.)
7. 왼쪽 대화 히스토리 메뉴에 표시될 대화 목록의 타이틀은 "YYYY년 MM월 DD일 HH시MM분SS초" 형식으로 최초 저장한다.
7. 대화 목록의 대화를 마우스로 1-click하면 해당 대화에 해당하는 웹 뷰 세션 목록과 챗 프롬프트 버튼 선택 상태를 복원하여 연 뒤, 대화 히스토리 메뉴를 접힘 상태로 변경한다.
9. 대화 목록의 대화에 마우스 오버하면 대화 타이틀의 음영을 바꿔 마우스 오버되었음을 알려주고, 대화 제목의 오른쪽 "..."으로 선택 가능한 클릭 가능한 영역을 표시하고, "..." 을 눌렀을때 컨텍스트 메뉴로
"Change Conversation Title"과 "Delete" (붉은 색 배경, 흰 색 글씨)를 표시한다.
10. "Change Conversation Title"을 선택하면 대화 타이틀이 변경할 수 있는 입력 폼으로 표시되고, 수정한 후, Enter 키를 눌렀을때, 해당 대화의 타이틀이 변경된다.
11. "Delete"을 선택하면 해당 대화가 삭제된다. 이 때, 삭제 여부를 확인하는 모달 팝업 메뉴가 표시되고, 삭제 선택 시 삭제되고 삭제되었음을 알려주고 모달 팝업이 닫히며, 취소 선택 시 모달 팝업이 닫힌다.
12. 왼쪽 대화 히스토리 목록은 마우스를 휠로 휠링하여 스크롤이 가능하며, 스크롤시 대화 목록이 이동한다. 이때 초기 가져오지 않은 대화목록이 있을 경우, 조회하여 추가된다.
13. 대화 히스토리는 chrome 브라우저의 indexeddb를 활용하여 데이터를 저장한다.
14. 앱이 개발되고 기능이 변경될 수 있어 IndexedDB 스키마 변경 사항이 있을 경우, 기존 DB의 데이터를 마이그레이션 할 수 있도록 구현한다.
