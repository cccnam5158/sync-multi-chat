**1. 문제/상황 이해**

(pain point #1)
프롬프트를 카테고리로 묶어서 관리하는 기능이 없어서 연관성 있는 프롬프트를 묶어서 관리할 수 없다.

(pain point #2)
"/" 커맨드에 모든 프롬프트가 출력되어 내가 자주 사용하는 프롬프트를 찾는데 시간이 걸린다. 그리고 "/" 입력 후 후속 입력 텍스트에 따라 필터링 되기는 하지만 컨텍스트 메뉴에 내 텍스트 입력이 가려져서 잘 인지하기 힘들다

(pain point #3)
sync multi chat을 이용해서 얼마나 많은 대화가 있었는지 확인하기 힘들고, 해당 대화 내역으로 이동하기 힘들다.

**2. 핵심문제와 가치가설 정의**

프롬프트를 카테고리로 묶어서 관리할 수 있도록 하고, 즐겨찾기한 프롬프트를 "/" 커맨드 상단에 노출시켜 편의성을 증가한다.

**3. 사용자를 위한 UX 개선안**

- 4-1. 왼쪽 메뉴 재 정리
	- (현재) Chat History
	- (개선) + New Chat, Dashboard, Prompt Hub, Chat History (w/Chat History Title Search)
	- (왼쪽 메뉴 항목 별 기획의도 - 왼쪽 메뉴 구성 변경 의도)
		- 1. "+New Chat" 
			- 메인 인풋 프롬프트 영역의 "New Chat"과 같은 기능을 한다. 즉, 해당 버튼이 클릭되면 새로운 Chat History Item이 생성되고, 모든 웹뷰는 새로운 챗을 시작할 수 있는 URL로 이동한다.
		- 2. "DashBoard" : Sync Multi Chat을 이용하는 사용자의 전반적인 이용 현황을 확인할 수 있는 대시보드로 구성한다.
			- 현재 선택 모드 : Single / Multi Mode - 선택된 AI 서비스 종류
			- 저장된 프롬프트 통계 : 카테고리 OO개, 전체 프롬프트 OO개
			- 대화 이력 : Chat History에 저장된 총 챗 히스토리 건수, 날짜별 대화 세션 저장 건수 차트 (연/월/일 선택 기능)
		- 3. Prompt Hub
			- Prompt Hub는 1x2로 구성되며, 1영역 (상단영역)과 하단 2영역은 두개 영역으로 나누어 왼쪽은 필터 영역, 오른쪽은 카테코리 영역으로 나뉜다.
			- 상단 영역의 왼쪽은 Prompt Hub (총 프롬프트 개수)가 배치되고, 오른쪽은 "+New Prompt" 버튼이 배치된다.
			- "+New Prompt" 버튼이 클릭되면 새로운 프롬프트를 Edit할 수 있는 창을 연다. 현재 Add Custom Prompt의 창과 같다. 이 창도 아래 기능 개선에 따라 수정이 필요하며 수정된 창이 뜬다.
			- 왼쪽 영역은 "필터" 기능
			  - 프롬프트 명으로 검색하기 위한 텍스트 입력 필드
			    - 텍스트 입력에 따라 해당 프롬프트가 필터링 되며, 해당 프롬프트가 존재하는 카테고리로 검색 범위가 좁아지도록 구성한다.
			  - 카테고리 선택 (최대 2-depth)
			  - 프롬프트에 지정된 태그 필터
			- 가운데 영역은 필터에 의해 조회된 프롬프트 목록이 조회된다.
			  - 보기 모드 : 테이블/미리보기
			  - 프롬프트 항목은 프롬프트 명, 간략 설명, 프롬프트 일부 미리보기, 수정, 복사, 복제, 삭제, 바로 실행 버튼을 표시한다.
			    - 프롬프트 타이틀 또는 프롬프트 수정 버튼을 클릭하면, 프롬프트 Edit 창으로 전환된다.
			- Prompt Hub에서의 항목을 추가 입력받기 위해 기존 프롬프트 빌더의 화면 구조를 이들 추가 항목을 추가 입력하기 좋게 최적의 UX로 수정한다.
			- 즉 프롬프트 에디터화면에서는 프롬프트 카테고리를 선택할 수 있다.
			- 그리고 카테고리는 최대 2-depth까지 관리 가능하며 이를 편집할 수 있는 별도 모달 팝업을 제공한다.
			  (e.g. 카테고리 1-depth는 "모든 카테고리", "Design", "Business", "Image", "Marketing", "Learning & Skills" 등으로 구성할 수 있으며,
			  각 1- depth 하위에 카테고리를 추가/삭제 할 수 있다.)
			- 즐겨찾기 카테고리는 사용자가 수정할 수 없으며, 즐겨찾기에는 카테고리 전체를 추가할 수 있다.
			- 그리고 카테고리가 지정되지 않은 모든 프롬프트는 "미분류" 카테고리에 추가된다. (즉, 이번 기능 개선으로 인해 기존 모든 프롬프트는 "미분류"에 위치하게 된다.)
			- 프롬프트 목록에서 개별 혹은 복수를 선택하여 카테고리로 한번에 지저는 기능을 제공한다.
		- 4. Chat History는 왼쪽 메뉴의 depth가 조정되는 것이지 기존의 모든 기능을 유지한다. 상세 유의 사항은 아래를 참고한다.
	- (유의) 기존 Chat History의 복수 선택 및 하단의 모두 선택 (All), 선택 해제 (Clear), 선택 삭제 (Delete) 기능이 유사하게 유지되어야 한다. 그리고 Chat History 상단에 Chat History Title 검색 및 정렬 기능이 제공되어야 한다. 그리고, 오늘, 어제, 지난 주, 지난 달, 그 이전 등으로 날짜별 카테고리 구분이 되어야 한다.  

- 4-2. "/" 커맨드 입력 시 프롬프트 선택 기능 개선
	- (현재) 전체 프롬프트 목록이 "/" 커맨드 입력 키보드 커서에 드롭다운 형식으로 표시되나, "/" 커맨드 뒤에 입력되는 한글/영어에 따라 필터링은 적용되고 있음
	- (개선1) "/" 커맨드 입력 시 표출되는 드롭다운 형태의 프롬프트 선택 메뉴에 텍스트 입력 란을 추가해서 "/" 커맨드 입력 후 키보드 입력 위치를 해당 프롬프트 선택 메뉴 창에 추가된 프롬프트 타이틀 상단의 텍스트 입력 영역으로 이동해줘 이 때 텍트 입력에 따라 프롬프트 타이틀에 매칭되는 항목으로 필터링 되고 프롬프트 타일틀 텍스트에 입력 값과 매칭되는 영역에 음영을 라이브로 추가해줘 
	- (개선2) "/" 커맨드의 기능 개선 및 프롬프트 카테고리 관리 기능 추가에 따라서 아래를 수정한다. 
	  - 즐겨찾기 프롬프트 카테고리가 가장 상단에 위치한다.
