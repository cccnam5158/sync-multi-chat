Phase 2Power User

Role Relay Builder

Planner → Critic → Writer처럼 각 AI에게 역할을 명시적으로 부여해, 단순 병렬 비교가 아니라 단계별 협업을 구성하는 고급 편집 흐름입니다.

자유 배치 대신 route preset 복제 기반으로 진입해 n8n/Zapier 연상을 줄임
핵심 가치는 "AI 선택 + 강점 지정 + 단계 프롬프트"에 집중
GuideBuilder는 초반 surface가 아니라, Quick Run 가치가 납득된 뒤 들어오는 단계입니다.

v5에서 달라진 점

v4의 React+ReactFlow 외부 의존성을 완전히 제거해 단독 HTML로 동작
Builder를 순수 CSS/JS 기반 flow canvas로 구현해 안정적으로 렌더링
v1~v4 + ideation v3 + POC chat thread의 핵심 컨셉을 모두 통합
Guide Mode ON/OFF로 컨셉 설명 / 실제 앱 느낌 전환 가능

왜 자유 배치형이 아닌가

사용자가 화면을 보자마자 n8n이나 Zapier처럼 인식하면, 현재 제품의 핵심 가치인 "각 AI 서비스의 특장점 지정"보다 "복잡한 워크플로우 툴"이라는 인상이 먼저 생깁니다. Builder는 노드 편집기보다 AI 역할 배정 보드에 가깝습니다.

Route Presets

빈 캔버스 대신 route preset 복제로 시작합니다.

1. Prompt Template질문을 정리하고 공유 기준을 만든다.
2. AI Roles각 AI의 강점과 역할이 분명하게 노출된다.
3. Merge차이점과 합의안을 통합한다.
4. Deliver메인 입력창 재주입과 저장 액션으로 연결한다.

Validation Notes

노드 드래그와 자유 연결을 제거해 n8n/Zapier와 다른 인상을 유지합니다.
각 stage는 service, role, prompt, outputVar를 동시에 가지므로 AI 선택 이유가 드러납니다.
Builder는 그래프 편집기가 아니라 route preset을 조정하는 AI 역할 지정 보드입니다.
