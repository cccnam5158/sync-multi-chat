# Sync Multi Chat v0.8.2: Gemini 세션 안정성 대폭 개선 — 대화 유지 · 타이핑 감지 · 스크롤 복원

**작성일**: 2026-03-19  
**버전**: v0.8.2

**Sync Multi Chat** v0.8.2는 **Gemini 웹뷰의 유휴 새로고침(Idle Refresh) 동작을 전면 개선**하여, 멀티 패널 대화 컨텍스트가 끊기지 않도록 하는 데 집중한 릴리스입니다.

---

## 유휴 새로고침 시 대화 컨텍스트 유지

### 배경

Gemini 웹뷰는 세션 만료를 방지하기 위해 일정 시간(5분) 유휴 후 자동으로 새로고침됩니다. 기존에는 새로고침 시 **새 채팅 URL(`/app`)**로 이동하여 진행 중이던 대화가 사라지는 문제가 있었습니다.

### 변경 사항

- **현재 URL 유지 리로드**: 유휴 새로고침 시 `loadURL(/app)` 대신 `reload()`를 사용하여 현재 대화 URL을 그대로 다시 로드합니다.
- 멀티 패널에서 여러 Gemini 웹뷰를 사용 중이어도 각각 **현재 진행 중인 대화가 유지**됩니다.

---

## 프롬프트 입력 중 자동 일시정지 (Typing-aware Pause)

### 배경

하단 메인 프롬프트 입력 영역에서 텍스트를 작성하는 도중 Gemini 웹뷰가 새로고침되면 포커스가 빼앗겨 입력이 중단되는 UX 문제가 있었습니다.

### 변경 사항

- 메인 프롬프트 입력에서 **텍스트를 입력하면 Gemini 새로고침 타이머가 자동으로 일시정지**됩니다.
- 일시정지 상태에서는 **타이머가 깜빡이며** 사용자에게 알려줍니다.
- **3초간 입력이 없으면** 타이머가 자동으로 재개됩니다.
- 이를 통해 **프롬프트 작성 중에는 절대 Gemini 웹뷰가 리로드되지 않습니다.**

---

## 스크롤 위치 및 커서 위치 복원

### 변경 사항

- 대화 URL(`/app/<session_id>`)에서 유휴 새로고침이 발생하면 **스크롤 위치를 저장**하고 리로드 후 복원합니다.
- 메인 프롬프트 입력의 **커서(케럿) 위치와 텍스트 선택 범위**도 새로고침 전후로 유지됩니다.

---

## 버전별 변경 요약

| 구분   | 내용 |
|--------|------|
| **개선** | Gemini 유휴 새로고침 시 현재 대화 URL 그대로 리로드 (새 챗 이동 방지) |
| **개선** | 프롬프트 입력 중 Gemini 새로고침 타이머 자동 일시정지 (3초 debounce) |
| **개선** | 리로드 후 대화 스크롤 위치 및 메인 입력 커서 위치 복원 |
